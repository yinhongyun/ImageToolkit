# ImageToolkit · 产品与交互设计方案

> 定位：面向游戏/美术资产的**本地拆图工具**（正式产品，非 demo）  
> 核心场景：一张角色表/素材拼图 → 自动或手动拆出单图 → 背景透明 → 命名导出  
> 技术基线：Python 3.11+ · PySide6 · Pillow · OpenCV · NumPy

---

## 1. 产品目标

### 1.1 一句话

把「拼在一张图里的多个素材」快速、可控地拆成透明底单图，并支持预览确认与精细调节。

### 1.2 成功标准

| 标准 | 说明 |
|------|------|
| 快 | 打开图后 1 次点击可完成自动检测与预览 |
| 准 | 自动检测后可微调框；手动可点选/手动画框 |
| 看得见 | 预览同时展示裁切范围边缘与透明效果 |
| 可导出 | 可改名、改路径、批量导出；结果可复查 |
| 不迷路 | 主流程始终是：导入 → 识别/框选 → 预览 → 导出 |

### 1.3 非目标（首版不做）

- 复杂 AI 抠图（人像/毛发级）——纯色/近似纯色底优先
- 在线协作、云同步
- 精灵动画序列编辑、图集重新打包（可列为二期）
- 矢量 SVG 编辑

## 2. 用户画像与典型任务

| 用户 | 任务 |
|------|------|
| 独立游戏开发 | 角色三视图/道具表拆成透明 PNG 进引擎 |
| 美术/外包验收 | 快速检查一张表里有几块、尺寸是否合规 |
| 内容同学 | 批量拆素材并按命名规范导出 |

**主路径（Happy Path）**

1. 拖入或打开一张角色表  
2. 点「自动识别」→ 画布上出现彩色边框预览  
3. 检查列表，删多余框 / 微调漏检框  
4. 看透明预览无误 → 设置导出目录与命名 → 导出  

**次路径（精确控制）**

1. 切到「手动」  
2. 鼠标移到某个角色上 → 自动吸附包围盒  
3. 拖拽手柄微调，或手动画矩形  
4. 确认该项 → 继续下一项 → 导出

### 4.1 视觉风格（易用、不花哨）

- **深色工作台**（#1E1E1E 底 + #2D2D2D 面板）：长时间看图不刺眼，透明棋盘格更清晰  
- **强调色**：单一青色/琥珀作选中与主按钮，避免彩虹乱标  
- **切片边框色**：自动分配低饱和区分色（第 1、2、3…），选中项加粗 + 白色手柄  
- **字体**：界面用系统 UI 字体；画布标注用等宽小号显示尺寸  
- **密度**：控件分组明确，左侧参数用「卡片分组」，右侧列表行高适中可点选  

不做成网页营销风；做成**可每天用的编辑器**。

## 5. 交互设计（详细）

### 5.1 导入

| 方式 | 行为 |
|------|------|
| 菜单/工具栏「打开」 | 系统文件对话框，过滤常见图片格式 |
| 拖放到窗口/画布 | 接受单张图片；若已有文档，弹确认「替换当前工作？」 |
| 最近文件 | 文件菜单列出最近 8 个路径 |

打开后：居中适配窗口（Fit），清空旧切片，提示「点自动识别，或切换手动」。

### 5.2 画布基础操作（始终可用）

| 操作 | 交互 |
|------|------|
| 缩放 | 滚轮；或 Ctrl+滚轮；工具栏百分比 |
| 平移 | 空格+拖；或中键拖 |
| 适应窗口 | 双击中键 / 快捷键 `Ctrl+0` |
| 实际像素 | `Ctrl+1` |

### 5.3 自动拆分

1. 用户点「自动识别」或快捷键 `A`  
2. 引擎：估背景色（四角采样）→ 色键/阈值 → 连通域 → 过滤过小区域 → 生成 Slice 列表  
3. 画布叠加所有切片边框；右侧列表同步  
4. 可选：识别后自动进入「选择」工具，选中第一项  

**参数（左侧，改完可点「重新检测」）**

| 参数 | 作用 | 默认思路 |
|------|------|----------|
| 背景容差 | 米色/白底容差 | 中等，实时提示「容差越大去底越多」 |
| 最小面积 | 过滤噪点 | 按图像面积百分比或绝对像素 |
| 边距 Padding | 包围盒外扩 | 2～8 px，防切到描边 |
| 合并间距 | 近距碎块合并 | 可选，默认关 |

**失败友好**

- 检测到 0 个：Toast + 建议「加大容差 / 改用手动画框」  
- 检测到过多碎块：提示「提高最小面积」并高亮过小项  

### 5.4 手动拆分（三种子交互）

#### 模式 A：点选吸附（默认手动模式）

1. 工具切到「选择/吸附」  
2. 鼠标在画布移动时，对指针下前景做局部连通域 → **虚线候选框**跟随  
3. 单击：候选框变为正式 Slice（实线）并选中  
4. 若指针下无前景：无候选，状态栏提示「未识别到内容，可改用画框」  

#### 模式 B：调节识别范围

选中 Slice 后：

- 8 向缩放手柄 + 边中点手柄  
- 拖动边/角调整矩形；Shift 锁定比例（可选）  
- 方向键微调 1px；Shift+方向键 10px  
- 右侧属性可直接改 X/Y/W/H 数值  

调节的是**导出裁切框**，透明处理仍基于框内像素相对背景的判定（见 §6）。

#### 模式 C：手动画框

1. 工具切到「画框」  
2. 按下拖拽画出矩形（实时橡皮筋）  
3. 松开：创建 Slice；若框内几乎无前景，警告但仍允许保留（用户可能要留白）  
4. 随后可进入调节手柄微调  

### 5.5 预览（「看得见范围」）

预览不是单独死板弹窗，而是**画布叠加 + 右侧明细预览**双通道：

| 预览层 | 开关 | 作用 |
|--------|------|------|
| 切片边框 | 默认开 | 彩色描边显示每个图像范围 |
| 尺寸标注 | 可选 | 框旁显示 `W×H` |
| 棋盘格底 | 可选 | 在透明预览时显示 |
| 透明效果 | 可选/选中时右侧常显 | 将该 Slice 去底后叠在棋盘格上 |

**推荐默认**

- 全局：显示所有边框  
- 选中项：加粗边框 + 右侧实时透明小预览  
- 工具栏一键「对比」：按住 `C` 临时隐藏边框看原图（像 Photoshop 按 H）  

### 5.6 切片列表与命名

- 列表项：色点 · 缩略图 · 名称 · 尺寸 · 勾选（是否导出）  
- 双击名称行内重命名；或在属性面板改  
- 默认名：`slice_01` 或基于源文件名 `goblin_01`  
- 支持「按规则重命名」：前缀 + 序号位数（导出前批量）  
- Delete 删除选中；可多选（Ctrl/Shift）  

### 5.7 透明处理（用户心智）

用户不需要懂算法，只理解：

> 「背景色会被去掉；角色本体保留。」

高级折叠面板（可选）：

- 背景色：自动（推荐）/ 吸管取色  
- 容差滑条  
- 去底模式：色键（默认）/ 仅边缘填充  

预览与导出使用**同一套参数**，避免「预览好看导出不对」。

### 5.8 导出

**导出面板字段**

- 目录：选择文件夹（记忆上次路径）  
- 格式：首版 PNG（透明必需）；后续可加 WebP  
- 命名：使用切片当前名称；冲突时弹窗「跳过 / 覆盖 / 自动后缀」  
- 范围：导出勾选项 / 仅当前 / 全部  

**导出确认流**

1. 点「导出…」打开轻量确认抽屉或对话框（列出将写出的文件名预览）  
2. 确认 → 进度条（多图时）→ 成功后「打开文件夹」按钮  

导出成功不强制关闭工程，便于继续改。

### 5.9 撤销 / 会话

- 可撤销：添加/删除/移动/缩放切片、重命名、参数应用前的切片集合快照  
- 不强制「工程文件」首版；二期可加 `.itk` 保存切片框  
- 关闭未导出变更时提示

## 6. 处理管线（与 UI 对齐）

```
源图 Document
    │
    ├─► 背景估计（四角/直方图） + 用户吸管覆盖
    │
    ├─► 自动识别 ──► Mask ──► Contours ──► Slice[]
    │
    ├─► 手动吸附 ──► 指针位置局部 Flood/Connected ──► 候选 Rect
    │
    ├─► 手动画框 ──► Rect 直接入 Slice[]
    │
    └─► 导出每个 Slice
            ├─ crop(rect + padding)
            ├─ apply transparency (同一 BackgroundParams)
            └─ save PNG + 使用 Slice.name
```

**性能**

- 预览用缩小代理图做检测（大图），导出用原分辨率  
- 悬停吸附做节流（约 30～50ms）+ 结果缓存  
- UI 线程与 OpenCV 计算用 `QThread`/`Worker`，防卡顿

## 9. 模块架构（实现分层）

```
src/
  app.py                 # 入口
  ui/
    main_window.py       # 主窗体布局
    canvas_view.py       # 画布、叠加、交互
    panels/
      tool_panel.py      # 左：工具+参数+预览开关
      slice_panel.py     # 右：列表+属性
      export_panel.py    # 右：导出
    dialogs/
      export_dialog.py
      settings_dialog.py
    widgets/
      checkerboard.py
      slice_item.py
  core/
    document.py          # Document / Slice 模型
    detector.py          # 自动识别、悬停吸附
    transparency.py      # 去底
    exporter.py          # 导出
    commands.py          # 撤销命令栈
  services/
    recent_files.py
    settings_store.py    # QSettings：路径、窗口、参数记忆
  resources/
    icons/               # 工具栏图标（简约线标）
    styles/
      app.qss            # 统一样式表
```

原则：

- **core 不依赖 Qt**（便于单测与批处理）  
- **ui 只编排**，算法全走 core  
- 样式集中 QSS，避免控件内散落魔法色值

## 14. 已确认决策

| 项 | 决定 |
|----|------|
| 产品名 | **ImageToolkit** |
| 导出格式 | **多格式支持，PNG 优先**（首版：PNG 必做；JPEG/WebP 等同期或紧随，JPEG 导出时提示无透明） |
| 主题 | **深色** |
| 工程文件 `.itk` | **首版包含**：保存/打开切片框、检测与去底参数、源图路径 |

### `.itk` 是什么？

`.itk`（ImageToolkit 工程文件）是**可选的工作存档**，不是导出的图片。

它大致会保存：

- 源图路径（或嵌入缩略信息）
- 你画好/识别好的所有切片框（位置、大小、名称、是否导出）
- 去底参数（容差、背景色等）
- 上次导出目录等偏好

用途：关掉软件后，下次打开同一个 `.itk`，框和名字都还在，不用重新识别。  
**不保存 `.itk` 也不影响正常使用**——打开图 → 拆 → 直接导出 PNG 完全够用。所以首版先不做，把拆图和导出做稳再加。
